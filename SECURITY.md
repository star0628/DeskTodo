# Security Policy

## Supported Versions

Security fixes are applied to the current `0.2.x` development line. Older personal-release builds may receive a fix only when the issue also affects the current line.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability or include private Todo data in an issue, screenshot, log, or test fixture.

Use GitHub's private vulnerability reporting page:

https://github.com/star0628/DeskTodo/security/advisories/new

Include the affected version, Windows version, reproduction steps, expected impact, and the smallest safe proof of concept. Remove task titles, local Store contents, usernames, and filesystem paths unless they are essential to understanding the issue.

The maintainer will acknowledge a complete report when practical, investigate it privately, and coordinate disclosure after a fix is available.
